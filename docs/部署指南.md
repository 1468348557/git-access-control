# GitLab 部署指南

## 架构概览

门禁脚本和 CI 配置文件托管在 HTTP 服务器 `http://21.73.16.160:8080/hobo-git-gate/` 上，通过 Python http.server 提供。

业务项目通过 `ci_config_path` 指向服务器上的 `.gitlab-ci.yml`，CI 运行时 curl 下载脚本，无需在业务项目中放任何门禁文件。

---

## 1. 前置条件

### 1.1 HTTP 服务搭建

在 Runner 可访问的服务器上，以 `hobo` 用户操作：

```bash
# 将门禁仓库 clone 到服务器
git clone git@gitlab.spdb.com:zh-1087/hobo-git-gate.git /home/hobo/hobo-git-gate

# 启动 HTTP 服务
bash /home/hobo/hobo-git-gate/server/start-http.sh
```

目录结构：

```
/home/hobo/hobo-git-gate/
├── server/
│   └── start-http.sh       # HTTP 启动脚本
├── template/
│   ├── .gitlab-ci.yml
│   └── scripts/
│       ├── git_gate.sh
│       └── approval.sh
├── docs/
└── ...
```

### 1.2 定时拉取 & 开机自启

以 `hobo` 用户配置 crontab（`crontab -e`）：

```cron
# 每 5 分钟拉取最新门禁脚本
*/5 * * * * cd /home/hobo/hobo-git-gate && git pull --rebase > /dev/null 2>&1

# 开机自动启动 HTTP 服务
@reboot bash /home/hobo/hobo-git-gate/server/start-http.sh
```

### 1.3 业务项目 Runner

每个业务项目需要注册一个 Shell Runner，tag 为 `shell`。

```bash
gitlab-runner register \
  --url http://192.168.8.143:8080/ \
  --registration-token <YOUR_REGISTRATION_TOKEN> \
  --executor shell \
  --tag-list shell \
  --description "shell-runner" \
  --non-interactive
```

## 2. 业务项目配置

### 2.1 设置 CI/CD 配置文件路径

**Settings → CI/CD → General pipelines → CI/CD configuration file**：

```
http://21.73.16.160:8080/hobo-git-gate/template/.gitlab-ci.yml
```

也可以通过 API 批量设置：

```bash
curl --header "PRIVATE-TOKEN: $TOKEN" \
  -X PUT "https://gitlab.spdb.com/api/v4/projects/$PROJECT_ID" \
  -H "Content-Type: application/json" \
  -d '{"ci_config_path": "http://21.73.16.160:8080/hobo-git-gate/template/.gitlab-ci.yml"}'
```

### 2.2 配置 CI/CD 变量

**Settings → CI/CD → Variables**，添加以下变量：

| 变量名 | 值 |
|---|---|
| `GATE_URL` | `http://21.73.16.160:8080/hobo-git-gate/template` |
| `GITLAB_PRIVATE_TOKEN` | Personal Access Token，scope 选 `api` |

### 2.3 分支保护配置

**Settings → Repository → Protected branches**：

| 分支 | 保护策略 |
|---|---|
| `master` | 不允许直接 push，仅允许 MR 合并 |
| `uat`, `uat_*` | 不允许直接 push，仅允许 MR 合并 |

**Settings → Merge requests → Merge checks**：开启 "Pipelines must succeed"。

定时任务每 5 分钟自动拉取，也可手动执行：

```bash
cd /home/hobo/hobo-git-gate && git pull --rebase
```

所有业务项目的下次 CI 自动使用最新脚本。

## 4. 流水线工作原理

### Stage 1: `gate`（自动执行）

Push 仅检查分支命名，MR 执行完整五项检查：

| 检查项 | Push | MR |
|---|---|---|
| 分支命名 | ✅ | ✅ |
| 合并方向 | — | ✅ |
| 变更行数 | — | ✅ |
| MR 标题 | — | ✅ |
| 基线校验 | — | ✅ |

变更行数 ≤ 100 行的 MR 由 `git_gate.sh` 直接调用 GitLab API 自动合并。> 100 行写入 artifact 传递给 Stage 2。

### Stage 2: `approval`（手动触发）

仅 > 100 行 MR 时出现，人工点击执行。审批通过后管理员在 GitLab 点击 Merge。
